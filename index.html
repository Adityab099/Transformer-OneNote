Session 0 — RNN, LSTM & GRU

0.1 Recurrent Neural Networks (RNN)

A Recurrent Neural Network (RNN) is a type of neural network designed to work with sequential data, where the order of information matters. It processes data step by step while keeping track of previous inputs using an internal memory called a hidden state. Unlike traditional feedforward networks, RNNs have connections that loop back, allowing past information to influence current and future outputs.

Recurrent Neural Networks (RNNs) are commonly applied when the sequence and order of data are important and the information changes over time. They are well-suited for tasks such as processing text, understanding speech, analyzing time-series data, and working with sensor readings.

Key Components
  • Hidden State (h_t): Stores information from previous steps and updates with each new input
  • Weights: Shared across time steps, making RNNs efficient for sequence learning
  • Activation Function: Typically tanh or ReLU to introduce non-linearity
h_t = f(∑ X_t · W_y + b_t)
Types of RNN by Input-Output Structure
Type Structure Example
One-to-One Single input → single output Image classification (baseline, not truly RNN)
One-to-Many Single input → sequence output Image captioning, music generation
Many-to-One Sequence input → single output Sentiment analysis, spam detection
Many-to-Many (same) Sequence → sequence (equal length) POS tagging, NER
Many-to-Many (diff) Sequence → sequence (different length) Machine translation (Encoder-Decoder)
Mathematics of Vanilla RNN
h_t = tanh(W_xh · x_t + W_hh · h_{t-1} + b_h)
y_t = W_hy · h_t + b_y

⚠️ Problem: Vanilla RNNs face the vanishing gradient problem, where gradients become extremely small as they are backpropagated through many time steps, making it difficult to learn long-range dependencies. They can also experience exploding gradients, where gradients grow too large and lead to unstable training.

Q: Suppose we have 3 input nodes, 2 hidden nodes, and 1 output node. Find total weights for RNN vs ANN.
RNN: 12 weights (shared weights across time steps add recurrent connections)
ANN: 8 weights (direct feedforward connections only)

0.2 LSTM — Long Short-Term Memory

LSTM (Long Short-Term Memory) is an improved Version of RNN introduced by Hochreiter and Schmidhuber. It is designed to learn long-term dependencies in sequential data using a special structure called a memory cell, which is regulated by three gates. Because of this ability, LSTMs are widely used in applications such as language translation, speech recognition, and time-series forecasting.

The Three Gates

1. Forget Gate — Decides what information to discard from the cell state.

f_t = σ(W_f · [h_{t-1}, x_t] + b_f)

Output ranges [0, 1]: 0 = forget completely, 1 = keep everything.

2. Input Gate — Controls what new information is added to the cell state.

i_t = σ(W_i · [h_{t-1}, x_t] + b_i)
C̃_t = tanh(W_c · [h_{t-1}, x_t] + b_c)
C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t

3. Output Gate — Determines the hidden state output for this time step.

o_t = σ(W_o · [h_{t-1}, x_t] + b_o)
h_t = o_t ⊙ tanh(C_t)

💡 Key Insight: The cell state Ct functions as a stable information pathway that allows gradients to propagate across long sequences with minimal degradation. This controlled flow helps mitigate the vanishing gradient issue in deep recurrent structures. The symbol ⊙ represents element-wise multiplication, where operations are performed independently across corresponding components of vectors or matrices.

0.3 GRU — Gated Recurrent Unit

GRU (Gated Recurrent Unit), introduced by Cho et al. in 2014, is a streamlined variant of recurrent neural networks that reduces architectural complexity compared to LSTMs. It eliminates the explicit cell state and consolidates control into two gating mechanisms: the update gate and the reset gate, which together regulate information retention and flow across time steps.

Gate Equations
Update gate: z_t = σ(W_z · [h_{t-1}, x_t])
Reset gate: r_t = σ(W_r · [h_{t-1}, x_t])
Candidate: h̃_t = tanh(W · [x_t, r_t · h_{t-1}])
Hidden state: h_t = (1 − z_t) · h_{t-1} + z_t · h̃_t
GRU vs LSTM Comparison
Feature LSTM GRU
Gates 3 (Input, Forget, Output) 2 (Update, Reset)
Cell State Yes (separate) No (hidden state only)
Training Speed Slower Faster
Parameters More Fewer
Performance Better for long-term memory Similar quality, less complexity

📝 Summary: Recurrent neural networks are capable of handling sequential inputs but struggle to retain information over long intervals. LSTMs address this limitation through a gated memory mechanism that improves long-range dependency learning. GRUs provide a more compact alternative with comparable performance and reduced computational complexity. These architectures formed the foundation of sequence modeling until the introduction of Transformer models in 2017, which shifted the paradigm in modern deep learning.

Session 1 — Introduction to Transformer Learning

Evolution in Deep Learning

The journey of deep learning follows a clear evolution:

  • ANN — Primarily used for structured/tabular data tasks
  • CNN — Designed for visual data such as images and spatial feature extraction
  • RNN → LSTM → GRU — Architectures specialized for sequential and time-dependent data, widely used before 2017
  • Transformers (2017) — A major breakthrough in deep learning introduced by Ashish Vaswani and team at Google through the paper "Attention Is All You Need", redefining sequence modeling using attention mechanisms

💡 Why Transformers? Traditional RNN-based models process input sequentially, meaning each token is handled one after another. This inherently limits parallel computation and makes it difficult to capture long-distance relationships effectively. In contrast, Transformers leverage self-attention mechanisms to evaluate all tokens in a sequence simultaneously, enabling efficient parallel processing and direct modeling of relationships between distant elements, regardless of their position in the sequence.

Core Concept

Transformers are built around the idea of sequence-to-sequence modeling using self-attention as the primary mechanism, removing the need for recurrence entirely. This framework underpins modern architectures such as GPT and BERT, and is widely applied in tasks like machine translation and question answering systems.

High-Level Architecture

Most state-of-the-art sequence modeling systems are structured in an encoder-decoder format:

  • Encoder: Converts an input sequence (x₁, …, xₙ) into a set of contextual representations z = (z₁, …, zₙ)
  • Decoder: Uses the encoded representation z to produce an output sequence (y₁, …, yₘ) step by step in an auto-regressive manner, where each prediction depends on previously generated tokens

The Transformer architecture is composed of stacked layers of self-attention modules and position-wise feed-forward networks in both encoder and decoder blocks, completely eliminating recurrence and convolution operations.

📝 Key Insight: Unlike RNN-based models that require sequential processing and limit parallel computation, Transformers enable full parallelization during training. Self-attention allows the model to directly capture relationships between all tokens in a sequence, regardless of their distance, which is a key advancement over earlier architectures.

Session 2 — Encoder–Decoder Architecture

Understanding the Encoder–Decoder Framework

The encoder–decoder architecture is primarily used for problems where the input and output sequences differ in length, such as machine translation, text summarization, image captioning, and speech-to-text conversion.

Encoder Module

The encoder transforms the input sequence into a structured internal representation through two key components:

  • Self-Attention Mechanism: Enables each token to dynamically attend to other tokens in the input, capturing contextual relationships across the sequence.
  • Feed-Forward Network: Applies non-linear transformations to refine representations and extract higher-level patterns.
Decoder Module

The decoder generates the output sequence using the encoded representation and includes three main components:

  • Masked Self-Attention: Processes previously generated tokens while preventing access to future positions during training.
  • Cross-Attention: Aligns decoder states with relevant encoder outputs to incorporate input context effectively.
  • Feed-Forward Network: Produces final token-level predictions.

End-to-End Workflow

  1. Tokenization: Input text is segmented into tokens (e.g., "I am learning AI" → ["I", "am", "learning", "AI"]).
  2. Embedding Layer: Tokens are mapped into dense vector representations.
  3. Encoding Stage: Self-attention layers build contextualized representations capturing global dependencies.
  4. Context Formation: The encoder output acts as a comprehensive representation of the input sequence.
  5. Decoding Stage: The decoder generates output tokens sequentially, using both self-attention and cross-attention.
  6. Final Output: Each predicted token depends on previously generated tokens and encoded input context.
⚠️ Key Limitation: Earlier encoder–decoder models compress the entire input into a single fixed-length representation, which leads to an information bottleneck when handling long sequences. This limitation was a key motivation for introducing attention-based architectures.
Q: How is the BLEU score used in evaluating sequence-to-sequence models?
BLEU (Bilingual Evaluation Understudy) is an evaluation metric that measures the similarity between generated text and reference outputs using n-gram overlap. It calculates precision over different n-gram levels, applies a brevity penalty for overly short outputs, and produces a score between 0 and 1, where higher values indicate better translation quality.

Session 3 — Self-Attention & Cross-Attention

3.1 Self-Attention Mechanism

Self-attention is the conversion of static context vectors into dynamic context vectors. It provides a dynamic approach to generating word representations by considering the entire context of the input sequence.

Why Self-Attention?

Traditional vectorization methods (One-Hot, Bag of Words, TF-IDF) treated each word as an isolated entity. Even word embeddings (Word2Vec, GloVe) are static — the same word always gets the same vector regardless of context.

📝 Example: The word "light" would have the same vector in "light as a feather" and "turn on the light." Self-attention produces different vectors for "light" in each context by weighing the surrounding words differently.
Components: Query, Key & Value
  • Query (Q): The current word asking "How relevant is every other word to me?"
  • Key (K): Reference points for all words that queries are compared against
  • Value (V): The actual contextual information/meaning of each word
Step-by-Step Computation
  1. Step 1 — Compute Q, K, V: Multiply input matrix X by learned weight matrices W_Q, W_K, W_V
  2. Step 2 — Calculate Attention Scores: Dot product of Q with K transposed → similarity matrix
  3. Step 3 — Scale: Divide scores by √d_k to prevent large variances that destabilize training
  4. Step 4 — Softmax: Normalize scores into a probability distribution (all weights sum to 1)
  5. Step 5 — Weight Values: Multiply softmax weights by V → contextually enriched output vectors
Attention(Q, K, V) = softmax(Q · K^T / √d_k) · V

3.2 Cross-Attention Mechanism

Cross-attention serves as an interface between the encoder and decoder components. It enables the decoder to selectively focus on relevant parts of the encoded input representation while generating each output token, effectively aligning input–output relationships.

Key Distinction from Self-Attention
  • Query (Q): Derived from the decoder’s intermediate representations
  • Key (K) and Value (V): Obtained from the encoder’s output representations

Although the computation follows the same attention formulation as self-attention, the critical difference lies in the origin of Q, K, and V, which enables interaction between encoded input context and generated output sequence.

Complexity
Metric Complexity Explanation
Time O(n² · d) Each word compared with every other word (n×n matrix), each comparison costs d operations
Space O(n²) Must store the full n×n attention matrix
Cross-Attention vs Self-Attention
Aspect Cross-Attention Self-Attention
Definition Decoder uses information from encoder Each word focuses on other words in the same sequence
Data Flow Between two different parts (encoder ↔ decoder) Within the same sequence (internal)
Focus Picking useful info from another part of the model Understanding how words in the same sentence relate
Example Translation: decoder selects right words from encoder Text analysis: understanding word connections in a sentence
Applications of Cross-Attention
  • Machine Translation: Aligns source words with target words
  • Speech Recognition: Matches audio sounds with text
  • Question Answering: Focuses on relevant text for the question
  • Text Summarization: Identifies the most important parts of input

Session 4 — Single-Head vs Multi-Head Attention

Single-Head Attention

Single-head attention applies one set of Query, Key, and Value projections to compute attention over the input sequence. It produces a weighted aggregation of token representations based on similarity scores, which are normalized using softmax to determine relative importance.

In some hybrid architectures such as SHA-RNN, single-head attention is integrated with recurrent units like LSTMs and can operate effectively with high-dimensional representations (e.g., 1024-dimensional embeddings), offering a simpler yet competitive alternative in certain settings.

📝 Scalability Issue: Attention mechanisms require computing dense score matrices whose size grows quadratically with sequence length. For long inputs (e.g., 4096 tokens), this results in extremely large 4096×4096 attention maps, motivating research into more efficient variants.

Multi-Head Attention

Multi-head attention enhances representational power by projecting the input into multiple subspaces and performing attention independently across each one. This allows the model to capture diverse relationships in parallel.

Operational Steps
  1. Projection Phase: Input X is linearly transformed into multiple sets of Q, K, and V matrices using separate learned weights
  2. Parallel Attention: Each head computes scaled dot-product attention independently
  3. Feature Fusion: Outputs from all heads are concatenated into a unified representation
  4. Output Mapping: A final linear transformation projects the concatenated result into the model dimension
MultiHead(Q, K, V) = Concat(head₁, head₂, ..., head_h) · WO

headi = Attention(Q·WiQ, K·WiK, V·WiV)
Advantages of Multi-Head Design
  • Diverse representation learning: Each head captures different linguistic or structural patterns
  • Parallel computation: Enables simultaneous learning of multiple attention views
  • Improved generalization: Reduces dependency on a single attention distribution
Placement in Transformer Architecture
Module Function
Encoder Self-Attention Models relationships within the input sequence
Masked Decoder Self-Attention Restricts attention to past tokens for autoregressive generation
Encoder–Decoder Attention Aligns decoder representations with encoded input context
Real-World Applications
  • Natural Language Processing: Machine translation, summarization, conversational AI systems
  • Computer Vision: Vision Transformers for image classification and understanding
  • Speech Processing: Speech recognition systems such as modern end-to-end ASR models

Session 5 — Positional Encoding

Need for Positional Encoding

Unlike RNN-based models, Transformers process all tokens in a sequence in parallel. As a result, they do not inherently capture word order. Positional encoding is introduced to inject sequence-order information into token embeddings so that the model can distinguish between different positions in a sentence.

Illustrative Insight

Consider the sentence "The cat sat on the mat." Each token is mapped to an embedding vector (E₁ to E₆). Without positional information, reordering tokens such as "cat sat" and "sat cat" would produce identical representations. Positional encoding ensures that each token carries explicit information about its position in the sequence.

Sinusoidal Positional Encoding (Original Transformer Design)

The original Transformer architecture encodes positions using sine and cosine functions at varying frequencies:

PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

Here, pos denotes the token position, i represents the embedding dimension index, and d_model is the total embedding size.

💡 Design Rationale: Sinusoidal functions provide continuous and smooth positional representations. They allow the model to infer relative distances between tokens using linear operations and generalize to sequence lengths beyond those seen during training.
Implementation Example (PyTorch)
import torch
import numpy as np

def sinusoidal_positional_encoding(seq_len, dim):
    base = 10000
    positions = torch.arange(seq_len).unsqueeze(1)
    div_term = torch.exp(-np.log(base) * (torch.arange(0, dim, 2) / dim))

    pe = torch.zeros(seq_len, dim)
    pe[:, 0::2] = torch.sin(positions * div_term)
    pe[:, 1::2] = torch.cos(positions * div_term)
    return pe

sequence = sequence + sinusoidal_positional_encoding(512, 768)

Categorization of Positional Encoding Methods

Method Common Models Key Property
Sinusoidal Original Transformer Deterministic encoding with extrapolation capability
Learned BERT, GPT-2 Trainable embeddings, limited generalization beyond training length
RoPE (Rotary Encoding) LLaMA and modern LLMs Encodes position using rotation in vector space, preserving relative structure
Relative Position Encoding T5, MPT Focuses on token-to-token distance rather than absolute position
ALiBi Falcon Adds distance-based bias directly to attention scores
Learned Positional Encoding (GPT-style)
import torch.nn as nn

class LearnedPositionalEncoding(nn.Module):
    def __init__(self, max_len, dim):
        super().__init__()
        self.embedding = nn.Embedding(max_len, dim)

    def forward(self, x):
        positions = torch.arange(x.size(1), device=x.device)
        positions = positions.unsqueeze(0).expand(x.size(0), -1)
        return x + self.embedding(positions)
RoPE (Rotary Positional Encoding)

RoPE encodes positional information through rotational transformations in vector space, allowing the model to naturally capture relative positions between tokens.

  • Relative position can be derived through rotational relationships between vectors
  • Supports long-context generalization beyond training limits
  • Maintains vector magnitude stability, improving training robustness

Significance

  • Order Awareness: Enables models to distinguish token positions within a sequence
  • Improved Generalization: Supports variable-length input sequences
  • Structural Disambiguation: Prevents identical treatment of tokens with different positions

Key Limitations

  • Fixed-Length Constraints: Some methods are limited by predefined maximum sequence lengths
  • Limited Flexibility: Absolute positional methods may not adapt well to dynamic sequence shifts

Session 6 — Feed-Forward Networks in Transformers

Core Architecture

The Feed-Forward Network (FFN) is a fundamental sub-layer within each Transformer encoder and decoder block. It is applied independently to each token representation (position-wise), using identical parameters across all positions.

FFN(x) = W₂ · σ(W₁ · x + b₁) + b₂
Structural Components
  • First Linear Projection (W₁): Expands the input embedding dimension (commonly 4× expansion, e.g., 512 → 2048)
  • Non-Linear Activation (σ): Introduces non-linearity; commonly ReLU in early models, and GeLU or SwiGLU in modern architectures
  • Second Linear Projection (W₂): Projects the transformed representation back to the original model dimension

Position-Wise Computation

The FFN operates independently on each token vector, without interaction across positions. While self-attention captures relationships between tokens globally, the FFN refines each token representation individually using shared weights, enabling high parallel efficiency.

Comparison: Attention vs Feed-Forward Network

Aspect Multi-Head Attention Feed-Forward Network
Function Models contextual relationships between tokens Transforms individual token features
Interaction Scope Global (cross-token interactions) Local (per-token processing)
Interpretation Aggregates information from the entire sequence Refines internal representation of each token
Parameter Contribution Moderate share of total parameters Typically dominates model size (~majority of parameters)

Theoretical Perspective

  • Universal Approximation Property: Two-layer neural networks with non-linear activations can approximate complex continuous functions
  • Expansion Phase (W₁): Maps inputs into a higher-dimensional space, improving feature separability (conceptually similar to kernel methods)
  • Compression Phase (W₂): Reduces dimensionality while preserving learned transformations
  • Feature Memory Interpretation: The FFN can be viewed as a learned transformation bank, where intermediate activations encode feature detectors and outputs represent refined embeddings

Modern Architectural Variants

  • GeLU Activation: Smooth non-linearity that improves gradient flow compared to ReLU
  • GLU / SwiGLU: Gated architectures that dynamically control information flow
  • Mixture of Experts (MoE): Replaces a single FFN with multiple expert networks, activating only a subset per token for efficient scaling
📝 Residual Design: The FFN is integrated with residual connections and layer normalization, ensuring stable gradient propagation and enabling deep stacking of Transformer layers without degradation in training performance.

Session 7 — Masking in Transformers

Definition of Masking

Masking is a control mechanism in Transformer architectures that determines which tokens are allowed to influence others during attention computation. It ensures that the model does not access irrelevant or future information, preserving correct training and inference behavior, especially in autoregressive generation tasks.

Major Types of Masks

1. Padding Mask

Padding masks are used when input sequences within a batch have varying lengths. Shorter sequences are padded to match the longest sequence, and the padding mask ensures these artificial tokens do not contribute to attention computation.

  • Applied in both encoder and decoder layers
  • Attention scores corresponding to padding tokens are set to −∞ before softmax
  • This forces their attention weights to become zero after normalization
2. Causal (Look-Ahead) Mask

Causal masking is used in the decoder self-attention mechanism to enforce autoregressive behavior. It prevents a token from attending to future tokens during training and inference.

  • Implemented using a triangular mask structure with −∞ values above the diagonal
  • Each position i can only attend to positions ≤ i
  • Prevents information leakage from future tokens during training
3. Attention Dropout Mask

Attention dropout randomly disables certain attention connections during training to improve generalization. This encourages the model to distribute attention more robustly rather than relying on specific token dependencies.

💡 Key Insight: Masking introduces structural constraints into the attention mechanism. Encoder attention remains fully bidirectional, while decoder attention is strictly causal. This design enables the model to understand full context during encoding while generating outputs sequentially in decoding.

Session 8 — Fundamental Learning Principles

1. Hebbian Learning Principle

Hebbian learning describes how neural connections strengthen through repeated simultaneous activation. In simple terms, when two neurons activate together frequently, the connection between them becomes stronger, forming the basis of learning and memory formation in neural systems.

Δw = η · x · y

Where x represents input activity, y represents output activity, and η is the learning rate.

import numpy as np

def hebbian_learning(inputs, outputs, lr=0.1):
    weights = np.zeros(inputs.shape[1])
    for x, y in zip(inputs, outputs):
        weights += lr * x * y
    return weights

X = np.array([[1, 0], [0, 1], [1, 1]])
Y = np.array([1, -1, 1])
weights = hebbian_learning(X, Y)
print("Learned Weights:", weights)

2. Associative Learning Principle

This principle models the relationship between stimuli and responses. It strengthens associations between paired inputs based on repeated co-occurrence, forming the foundation of classical associative learning systems.

def associative_learning(pairs, lr=0.1):
    association_map = {}
    for stimulus, response in pairs:
        association_map[stimulus] = association_map.get(stimulus, 0) + lr * response
    return association_map

pairs = [("bell", 1), ("food", 1), ("bell", 1)]
result = associative_learning(pairs)
print("Associations:", result)

3. Correlation-Based Learning Principle

This learning rule updates weights based on statistical correlation between inputs and outputs. It focuses on how deviations from the mean of both variables influence learning strength.

Δw = η · (x − x̄)(y − ȳ)
def correlative_learning(inputs, outputs, lr=0.1):
    weights = np.zeros(inputs.shape[1])
    mean_x = np.mean(inputs, axis=0)
    mean_y = np.mean(outputs)

    for x, y in zip(inputs, outputs):
        weights += lr * (x - mean_x) * (y - mean_y)
    return weights

Vanishing Gradient Challenge

During backpropagation, gradients are repeatedly multiplied as they flow through layers. When activation functions compress values into small ranges (such as sigmoid or tanh), gradients shrink rapidly, leading to exponential decay. As a result, early layers receive extremely weak updates, preventing effective learning in deep networks.

This issue often leads to training instability, slow convergence, and poor performance in deep architectures compared to shallower models.

Session 9 — Optimization: Gradient Descent & Adam Optimizer

Understanding Gradient Descent

Gradient Descent is a fundamental optimization technique used to minimize a loss function by iteratively updating model parameters in the direction of the negative gradient. Intuitively, it can be viewed as navigating downhill on a loss landscape to reach a point of minimum error.

θt+1 = θt − η · ∇L(θt)

Here, η denotes the learning rate, and ∇L(θ) represents the gradient of the loss function with respect to parameters θ.

Variants of Gradient Descent
Type Data Utilization Behavior Use Case
Batch Gradient Descent Full dataset Stable and deterministic updates Small-scale datasets requiring precise convergence
Stochastic Gradient Descent (SGD) Single sample per update Noisy but fast convergence behavior Online learning and large-scale optimization
Mini-Batch Gradient Descent Subset of data (e.g., 32–512 samples) Balanced trade-off between stability and efficiency Standard choice in deep learning frameworks
Challenges in Vanilla Gradient Descent
  • Uniform learning rate across all parameters leads to inefficient updates in complex loss landscapes
  • Highly sensitive to learning rate selection, requiring careful tuning
  • Susceptible to saddle points, which are more prevalent than local minima in high-dimensional spaces
Basic Implementation Example
import numpy as np

def gradient_descent(start, lr=0.1, iterations=50, tolerance=1e-6):
    x = start
    for i in range(iterations):
        gradient = 2 * x  # derivative of x²
        updated_x = x - lr * gradient

        if abs(updated_x - x) < tolerance:
            break

        x = updated_x

    return x

result = gradient_descent(5.0)
print("Optimized value:", result)

Adam Optimizer (Adaptive Moment Estimation)

Adam is an advanced optimization algorithm that integrates momentum-based updates and adaptive learning rates. It combines first-order (mean) and second-order (variance) moment estimates to improve convergence stability and speed.

Core Computation Steps
mt = β₁ mt−1 + (1 − β₁) gt
vt = β₂ vt−1 + (1 − β₂) gt²
m̂t = mt / (1 − β₁ᵗ), v̂t = vt / (1 − β₂ᵗ)
θt+1 = θt − η · m̂t / (√v̂t + ε)

Typical hyperparameter values: β₁ = 0.9, β₂ = 0.999, ε ≈ 10⁻⁸

💡 Key Advantage: Adam dynamically adjusts learning rates for each parameter by combining momentum and variance tracking, resulting in faster convergence, reduced oscillations, and improved performance across diverse deep learning tasks.
Comparison of Advanced Optimizers
Optimizer Core Idea Primary Use Case
AdamW Decouples weight decay from gradient updates Modern deep learning and large language models
Adagrad Adaptive learning rate based on accumulated squared gradients Sparse feature spaces and embeddings
RMSprop Exponentially weighted moving average of squared gradients Non-stationary problems and recurrent networks
L-BFGS Approximate second-order optimization using limited memory Small-scale, high-precision optimization tasks

Session 10 — Loss Functions

Definition of Loss Function

A loss function quantifies the discrepancy between predicted outputs and actual target values. It provides a numerical measure of model error, guiding the optimization process during training by indicating how far predictions deviate from ground truth.

Loss Functions for Regression

1. Mean Squared Error (MSE)
MSE = (1/n) · Σ(yᵢ − ŷᵢ)²

Application: Standard regression problems.
Limitation: Highly sensitive to outliers due to squared error amplification.

2. Mean Absolute Error (MAE)
MAE = (1/n) · Σ|yᵢ − ŷᵢ|

Application: Robust regression scenarios.
Limitation: Gradient is not smooth at zero, which can slow optimization.

3. Huber Loss

Huber loss is a hybrid formulation that behaves like MSE for small errors and like MAE for large errors, controlled by a threshold parameter δ.

L = ½(y − ŷ)²   if |y − ŷ| ≤ δ
L = δ|y − ŷ| − ½δ²   otherwise

Loss Functions for Classification

4. Binary Cross-Entropy (Log Loss)
L = −[y·log(p) + (1 − y)·log(1 − p)]

Application: Binary classification tasks with sigmoid activation.
Note: Most widely used loss for two-class problems.

5. Categorical Cross-Entropy
L = −Σᵢ yᵢ · log(pᵢ)

Application: Multi-class classification with softmax outputs, where only one class is correct per sample.

6. Hinge Loss
L = max(0, 1 − y·ŷ)

Application: Margin-based classification methods such as Support Vector Machines.

📝 Selection Guide: Use MSE for general regression tasks, Huber loss for robustness against outliers, Binary Cross-Entropy for binary classification, and Categorical Cross-Entropy for multi-class problems.

Session 11 — Optimization Techniques

Classification of Optimization Methods

First-Order Methods (Gradient-Based)
  • Gradient Descent Variants: Includes SGD, Mini-Batch GD, and Momentum-based updates
  • Adaptive Optimizers: Adam, AdamW, RMSprop, Adagrad — adjust learning rates dynamically per parameter
Second-Order Methods (Gradient + Curvature Information)
  • Newton’s Method: θ_new = θ_old − H⁻¹ · ∇f(θ). Provides fast convergence but is computationally expensive due to Hessian inversion (O(n³)), making it impractical for large-scale neural networks
  • BFGS: Approximates the Hessian matrix using gradient information
  • L-BFGS: Memory-efficient variant that retains only limited historical gradient information
Stochastic-Based Optimization
  • Simulated Annealing: Introduces controlled randomness to escape local optima, gradually reducing exploration over time
  • Random Search: Simple but effective strategy for exploring high-dimensional hyperparameter spaces
Evolutionary Optimization Methods
  • Genetic Algorithms: Optimize solutions through selection, crossover, and mutation across a population of candidates
  • Differential Evolution: Generates new candidates using vector differences: a + F·(b − c)
Swarm Intelligence Methods
  • Particle Swarm Optimization (PSO): Agents update positions based on personal best and global best solutions
  • Ant Colony Optimization (ACO): Uses pheromone-inspired reinforcement to discover optimal paths

Learning Rate Scheduling Strategies

  • Cyclical Learning Rates (CLR): Learning rate oscillates between bounds to improve exploration and escape flat regions
  • One-Cycle Policy: Gradual increase followed by decrease in learning rate combined with inverse momentum scheduling, enabling faster convergence and improved generalization
  • Warm-up Strategy: Gradually increases learning rate at the beginning of training, commonly used in Transformer-based models for stable optimization

Challenges in High-Dimensional Optimization

⚠️ Saddle Point Dominance: In high-dimensional optimization landscapes, saddle points are significantly more prevalent than local minima. These points have zero gradients but mixed curvature, making optimization difficult. Most local minima lie close to the global minimum in deep learning loss surfaces.

Hyperparameter Optimization Strategies

  • Grid Search: Exhaustive evaluation of all parameter combinations, computationally expensive but systematic
  • Random Search: More efficient exploration of large search spaces with competitive performance
  • Bayesian Optimization: Uses probabilistic surrogate models (e.g., Gaussian Processes) to guide sampling efficiently in expensive search spaces

Session 12 — Regularization in Transformer Models

Transformer architectures possess high representational capacity, enabling them to learn complex patterns effectively. However, this also increases the risk of overfitting, particularly during fine-tuning on limited datasets.

1. Dropout Regularization

Dropout reduces overfitting by randomly deactivating a subset of neuron activations during training, preventing excessive reliance on specific features.

In Transformer architectures, dropout is commonly applied at multiple stages:

  • After token embedding layers
  • After outputs of attention and feed-forward sublayers
  • Within attention score computations prior to softmax normalization

Typical configuration: dropout rates generally range from 0.1 to 0.3, while large-scale models often use lower values (0.05–0.1) to maintain sufficient learning capacity.

2. Weight Decay (L2 Regularization)

Weight decay introduces a penalty proportional to the squared magnitude of model parameters, discouraging excessively large weights and improving generalization stability.

3. Label Smoothing

Label smoothing replaces hard one-hot targets with softened probability distributions, reducing model overconfidence and improving calibration.

For example, instead of a target like [0, 1], smoothed targets may resemble [0.05, 0.95].

4. Attention Dropout

This technique applies dropout directly to attention weights before softmax normalization, encouraging more distributed attention patterns across tokens.

5. Stochastic Depth

Stochastic depth randomly bypasses entire layers during training, effectively creating dynamic shallow networks within deep architectures. This improves gradient propagation and reduces overfitting in very deep Transformers.

📝 Key Insight: No single regularization technique is universally optimal. In practice, combinations such as dropout, label smoothing, and weight decay form the foundation of robust training, while advanced methods like stochastic depth further enhance performance in deeper models. Effective regularization ensures strong generalization rather than memorization of training data.

Session 13 — BERT (Bidirectional Encoder Representations from Transformers)

Overview of BERT

BERT is an encoder-only Transformer architecture designed to learn deep language representations by processing text in a fully bidirectional manner. Unlike sequential models, it analyzes the entire sentence context simultaneously, enabling richer contextual understanding of each token.

Pre-Training Objectives

BERT is trained on large-scale unlabeled corpora using two core self-supervised objectives:

1. Masked Language Modeling (MLM)
  • Randomly masks approximately 15% of input tokens
  • Model predicts original tokens using surrounding bidirectional context
  • Final hidden states are projected to vocabulary space via embedding matrix multiplication followed by softmax
  • Loss is computed only on masked positions
  • Improves deep contextual representation learning, though training converges slower than autoregressive approaches
2. Next Sentence Prediction (NSP)
  • Determines whether sentence B logically follows sentence A
  • Training data contains both true consecutive pairs and randomly sampled sentence pairs
  • The [CLS] token representation is used for binary classification (IsNext vs NotNext)
  • Encourages learning of inter-sentence semantic relationships
💡 Key Insight: MLM enhances token-level contextual understanding, while NSP captures discourse-level relationships between sentences. Together, they enable a holistic understanding of language structure.

Fine-Tuning Applications

  • Text Classification: Uses [CLS] token representation for tasks like sentiment analysis and spam detection
  • Question Answering: Predicts start and end positions of answer spans within a context
  • Named Entity Recognition (NER): Assigns token-level labels for entity extraction tasks
  • Machine Translation: Provides contextual embeddings for improved translation quality
  • Text Summarization: Supports generation of condensed representations of long-form text

📎 Appendix A — Activation Functions

Complete reference guide covering commonly used neural network activation functions

Activation Functions — Quick Reference Guide

Activation functions inject non-linearity into neural networks. Without them, multiple stacked layers behave like a single linear transformation, severely limiting model expressiveness.

Activation Mathematical Form Output Range Typical Usage Key Limitation
Linear f(x) = x (−∞, ∞) Regression output layer No non-linear transformation
Sigmoid σ(x) = 1 / (1 + e⁻ˣ) (0, 1) Binary classification outputs Gradient saturation; not zero-centered
Tanh (eˣ − e⁻ˣ) / (eˣ + e⁻ˣ) (−1, 1) Recurrent network hidden states Still prone to vanishing gradients
ReLU max(0, x) [0, ∞) Default hidden layer activation Dead neuron problem (inactive units)
Leaky ReLU max(αx, x), α ≈ 0.01 (−∞, ∞) Robust hidden layers, GANs Introduces small negative slope hyperparameter
Softmax eˣⁱ / Σⱼ eˣʲ (0, 1), sum = 1 Multi-class classification output Computationally sensitive to large logits
GeLU x · Φ(x) Approx. (−0.17, ∞) Transformer architectures Higher computational cost than ReLU
Why is ReLU preferred in hidden layers but avoided in output layers?
Hidden layers benefit from ReLU due to computational efficiency, strong gradient propagation for positive inputs, and sparse activation behavior, which improves learning dynamics.

Output layers, however, must align with the problem structure. ReLU is unsuitable for probabilistic outputs or bounded targets. Classification tasks require probability distributions (sigmoid for binary, softmax for multi-class), while regression tasks typically require unrestricted linear outputs.

📎 Appendix B — Backpropagation

Mechanism through which neural networks iteratively learn by correcting prediction errors

Backpropagation — End-to-End Learning Process

Step 1: Forward Computation

The input data flows through successive layers (input → hidden → output) to generate a prediction from the model.

Step 2: Error Evaluation

Error = Target Output − Model Prediction
Loss Function = Quantitative measure of prediction deviation (e.g., MSE for regression, Cross-Entropy for classification)

Step 3: Gradient Propagation (Chain Rule Application)

  1. Output Layer Sensitivity: Compute error gradient at output using (Prediction − Target) scaled by derivative of activation function
  2. Hidden Layer Contribution: Propagate error backward by aggregating gradients from downstream layers and multiplying by local activation derivatives
  3. Parameter Update Rule: Weights are adjusted using gradient descent: Δw = −η × gradient signal × input activation

Illustrative Example

Consider a feedforward network predicting binary student performance outcome:

  • Inputs: Study hours = 3 (scaled to 0.3), prior performance = 70 (scaled to 0.7)
  • Model Output: Predicted probability = 0.542
  • Ground Truth: Actual label = 1 (pass)
  • Error Computation: 0.542 − 1.0 = −0.458
  • Activation Sensitivity: Sigmoid derivative = 0.542 × (1 − 0.542) = 0.248
  • Gradient Signal: −0.458 × 0.248 = −0.1136
  • Learning Step: This gradient is propagated backward to update all connected weights proportionally
Key Insight: Backpropagation enables efficient credit assignment across multiple layers by systematically distributing error signals using the chain rule, allowing deep networks to optimize millions of parameters simultaneously.

📎 Appendix C — Probability & Distributions

Statistical foundations underlying machine learning and deep learning systems

Random Variables and Probability Distributions

Random Variables

  • Discrete Random Variables: Take countable outcomes (e.g., number of heads in coin flips: {0,1,2,...})
  • Continuous Random Variables: Take values over a continuous range (e.g., height, temperature, time)

Probability Mass Function (PMF) — Discrete Case

P(X = x) = p(x), where probabilities satisfy 0 ≤ p(x) ≤ 1 and the sum over all outcomes equals 1.

Probability Density Function (PDF) — Continuous Case

f(x) ≥ 0 with total integral equal to 1. Probabilities are computed over intervals:

P(a ≤ X ≤ b) = ∫ₐᵇ f(x) dx, while P(X = exact value) = 0 for continuous variables.

Skewness interpretation: Symmetric distributions have zero skewness. Positive skew indicates right-tailed distributions (e.g., income data), while negative skew indicates left-tailed distributions (e.g., easy-score distributions).

A stochastic process introduces randomness into system evolution over time. A Markov process is a special case where the future depends only on the present state.

P(X_{t+1} | X_t, X_{t-1}, ..., X_0) = P(X_{t+1} | X_t)

📎 ANN Basics & Practice Questions

Core theory + exam-oriented Q&A

ANN Basics — Q&A Bank

Q1: Explain hyperplanes mathematically.
A hyperplane is a flat subspace with one dimension less than its ambient space (e.g., a line in 2D, a plane in 3D).

In ℝⁿ: w₁x₁ + w₂x₂ + ... + wₙxₙ + b = 0 or in vector form: w·x + b = 0
• w = normal vector perpendicular to the hyperplane
• b = bias term (shifts the plane from origin)
• Divides space into two half-spaces: w·x + b > 0 and w·x + b < 0
• Distance from point x₀: |w·x₀ + b| / ||w||
• In ML: hyperplanes are decision boundaries in classifiers like SVM
Q2: Difference between normal and partial differentiation?
Normal: Single variable. dy/dx. Slope of tangent to a curve. Example: y = x³ → dy/dx = 3x².
Partial: Multiple variables. ∂z/∂x. Rate of change w.r.t. one variable while holding others constant. Example: z = x²y + sin(x) → ∂z/∂x = 2xy + cos(x), ∂z/∂y = x².
Q3: Difference between ANN and BNN?
ANN: Software/hardware implementation. Artificial neurons with weights and activation functions. Learns via backpropagation. Predefined architecture.
BNN: Natural brain network. Biological neurons and synapses. Electrochemical signals. Hebbian plasticity. Massive parallelism, low power, fault tolerant. Not fully understood.
Q4: What are the limitations of ANN?
  • High computational cost — requires GPUs/TPUs
  • Needs massive datasets — poor with small data
  • Black box nature — hard to interpret decisions
  • Overfitting — memorizes training data
  • Vanishing/exploding gradients in deep networks
  • Hyperparameter sensitivity — many settings to tune
  • Catastrophic forgetting — forgets old tasks when learning new ones
  • Adversarial vulnerability — small input changes can fool the network
  • Sequence length limitations (RNNs)
  • No common sense reasoning
Q5: What is the significance of bias?
Bias is like the intercept in y = mx + c. Without bias, every decision must pass through the origin (0,0). Bias allows the decision boundary to be shifted anywhere.

Analogy: In a spam filter, weights look at email words. Bias is the initial suspicion level before reading any words. A high bias means "assume spam unless proven otherwise."

Why important: Flexibility to fit data better. Prevents zero-output. Enables learning complex patterns.
Q6: Differentiate between linear and non-linear activation functions.
Linear: f(x) = x. No transformation. Stacking linear layers = single linear layer. Used only for regression output.
Non-linear: Enables deep networks to learn complex mappings. Types: Sigmoid (binary output), Tanh (RNN hidden), ReLU (default hidden layer), Leaky ReLU (fixes dying ReLU), Softmax (multi-class). Without non-linearities, networks cannot approximate arbitrary functions.
Q7: Explain classifications of AI through a Venn diagram structure.
Outermost: AI — machines mimicking human intelligence
  ├── ML: Systems learning from data without explicit programming
  │  ├── DL: Multi-layer neural networks learning representations
  │  └── RL: Agents learning by interacting with environments
  ├── Symbolic AI / Expert Systems (rule-based, outside ML)
  └── Application domains: Computer Vision, NLP, Robotics

By capability: Narrow AI (specific tasks, exists today) → General AI (human-like, not achieved) → Superintelligent AI (beyond human, hypothetical)

Practice Code — ANN Apple Classifier

import numpy as np

# Data: {color (0=green, 1=red, 2=yellow), weight in gm}
X = np.array([[0,120],[0,130],[1,140],[1,150],[2,160],[2,170]])
y = np.array([0, 0, 0, 1, 1, 1])  # Labels

def step_function(x):
    return 1 if x >= 0 else 0

weights = np.random.randn(2)
bias = 0.5
learning_rate = 0.05

for _ in range(120):
    for i in range(len(X)):
        linear_output = np.dot(X[i], weights) + bias
        y_pred = step_function(linear_output)
        error = y[i] - y_pred
        weights += learning_rate * error * X[i]
        bias += learning_rate * error

print("Final Weights:", weights)
print("Final Bias:", bias)

Practice Code — Simple RNN Sequence Predictor

import numpy as np

seq = [i for i in range(1, 11)]  # 1 to 10
X, y = [], []
for i in range(len(seq)-3):
    X.append(seq[i:i+3])
    y.append(seq[i+3])

predictions = []
for x in X:
    pred = 0.5*x[2] + 0.3*x[1] + 0.2*x[0]  # Recency-weighted
    predictions.append(round(pred, 1))

print(f"Predictions: {predictions}")  # [1.9, 2.8, 3.7, ...]
print(f"Actual:      {y}")            # [4, 5, 6, ...]
